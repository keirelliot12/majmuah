## 🎯 THE 5 GOLDEN RULES

**These are non-negotiable:**

```
1️⃣  DESIGN BEFORE CODE
    Always get spec approved first
    Never code blind

2️⃣  TEST-DRIVEN DEVELOPMENT
    Write test before implementation
    NO EXCEPTIONS

3️⃣  PLAN EVERYTHING
    Break into 2-5 minute tasks
    Each task: clear acceptance criteria

4️⃣  TASK-BASED EXECUTION
    One task at a time
    Sequential, verified progress

5️⃣  QUALITY NON-NEGOTIABLE
    100% tests passing
    Zero warnings
    Code reviewed
```

## 🔄 THE WORKFLOW

```
START
  ↓
BRAINSTORM (Ask questions)
  ↓
DESIGN (Create specification)
  ↓
APPROVAL (Get stakeholder sign-off)
  ↓
PLAN (Break into tasks)
  ↓
FOR EACH TASK:
  ├─ Write failing test (RED ❌)
  ├─ Watch test FAIL
  ├─ Write minimal code (GREEN ✅)
  ├─ Watch test PASS
  ├─ Refactor (REFACTOR ✅)
  ├─ Code review

  └─ Commit & move to next

ALL TESTS PASS ✅
  ↓
MERGE TO MAIN
  ↓
CELEBRATE 🎉
```
